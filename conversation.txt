CLAUDE:
Agreed -- the hardest moment is STARTING, and small goals lower the activation
barrier. But flagged the twist: the reward IS the addiction, which is powerful
but invites gaming/satisficing (minimum study for maximum internet). So the
principle is "small to start, then shaped": tiny early wins, then raise the bar;
frequent short cycles over one big block; diminishing returns + daily cap so
small goals don't become infinite internet; genuine recall so it's not just
clicking; visible progress so studying carries its own reward. Honest note: this
manages the pull (Premack principle), it's a scaffold not a cure. Asked the user
to define the atomic study unit (the smallest rewardable action).

================================================================================
KEY DECISIONS (quick reference)
================================================================================
- Concept: study-to-earn-internet. Study Platform is the product; Guardian is optional.
- Independence: study works with or without the Guardian; contract between them = a score.
- Architecture: Study Platform (Admin + Tutor + Exam) | Guardian (Reward Engine + Network Wall).
- Knowledge backend: NotebookLM or Gemini behind a swappable Knowledge-Source adapter; OCR offloaded.
- Atomic unit: a configurable-size exam. 3 min/question. Default 20 q (60 min); min 5 q (15 min).
- Quiz: multiple choice. Pass bar 70%. Post-fail cooldown ~10 min. Rotating bank. (Anti-gaming.)
- Reward: flat on pass for v1 (scaled = P1 upgrade). Daily cap (default 3 h).
- Network: WiFi-only. MikroTik Hotspot. Targeted & allow-by-default (only the student's device, by MAC).
- Override: parent, via FIDO2/WebAuthn, logged; register a backup key.
- Everything configurable, ships with sensible defaults.
- Build order (value-first): Phase 1 tutoring core (scores) -> Phase 2 tutor/knowledge depth -> Phase 3 Guardian.
